## SA 分析筆記

### 核心業務模組
1. **遊戲生命週期**：上架 → 標籤 → 推薦 → 排序 → 維護 → 下架
2. **金流閉環**：儲值（三方支付）→ 遊戲投注 → 提領（公會監控）
3. **風控體系**：禁用語 + 聊天監控 + 轉帳額度確認 + 操作 audit log
4. **營運工具**：活動 + 任務 + 紅包 + 內容發布（banner/跑馬燈/公告）
5. **權限管理**：RBAC（角色 → 帳號），後台操作全程記錄

### 與 CGP 對比（待補充）
- Nova 是新版平台，架構更現代（SPA + REST API）
- CGP 有 175 子功能，Nova 目前 43 個（仍在開發中）
- Nova 多了「公會」和「道具」系統，CGP 沒有

## 頁面欄位細節（SA 需求分析用）

### 遊戲維護設定
- **篩選條件**：遊戲名稱（搜尋）、狀態（下拉）、種類（下拉）、進階篩選
- **表格欄位**：序號、標籤、娛樂城、種類、遊戲名稱、狀態、VIP 等級以上、操作、備註
- **特殊功能**：全廳維護控制（批次變更所有遊戲維護狀態）
- **分頁 Tab**：金幣 / 星幣（雙幣種系統）
- **分頁**：每頁 20 筆

### 用戶查詢
- **篩選條件**：帳號（多帳號逗號分隔）、狀態（下拉）、所屬公會（下拉）、進階篩選
- **表格欄位**：暱稱、所屬公會、帳號、註冊來源、綁定手機、VIP、註冊時間（可排序）、最後登入（可排序）、狀態、操作
- **特殊功能**：批次操作、匯出報表
- **分頁**：每頁 20 筆

### 三方支付 — 供應商
- **篩選條件**：供應商名稱（搜尋）、狀態（啟用/停用）
- **表格欄位**：名稱、狀態（switch 開關）、供應商代碼、操作（編輯）
- **特殊功能**：新增供應商（僅支援金幣類型）
- **已有供應商**：遠傳電信、玉山銀行、台灣大哥大、中華電信、全支付、信用卡、星運測試商、zacktest 等 20 家
- **啟用中**：星運測試商（AB00888）、zacktest
- **分頁**：每頁 20 筆

## 系統設計特徵（SA 面試素材）

### 架構特點
1. **SPA 架構**：Vue/React 前端 + REST API 後端
2. **RBAC 權限**：角色 → 帳號，細粒度功能權限控制
3. **雙幣種系統**：金幣 + 星幣，遊戲管理按幣種分 tab
4. **多層金流**：供應商 → 支付方式 → 付款通道 → 儲值金額表
5. **Audit Trail**：帳號操作紀錄、公會操作記錄、交易紀錄全程追蹤
6. **風控機制**：禁用語 + 聊天監控 + 大聲公監控 + 轉帳額度確認

### 業務流程
1. **儲值流程**：用戶選方案 → 三方支付（供應商→通道→方式）→ 入帳
2. **遊戲流程**：上架設定 → 標籤/推薦/排序 → VIP 限制 → 維護管理
3. **公會流程**：建立 → 加入 → 貢獻 → 提領（需監控）
4. **營運流程**：活動設定 → 任務配置 → 紅包發放（需審核）→ 內容發布
