# Nova 後台篩選欄位權重與輸入限制規範

## 權重順序（由高到低）

### 1. 帳號 (Account)
- **權重**: 1（最高優先）
- **輸入限制**: 2-20 字元，英數字底線，可用半形逗號分隔多個帳號
- **驗證規則**: `^[a-zA-Z0-9_]{2,20}$`（單個帳號）
- **頁面**: 用戶查詢、會員支付查詢、交易紀錄查詢、投注紀錄查詢、帳號設定

### 2. 狀態 (Status)
- **權重**: 2
- **輸入限制**: 下拉選單（啟用/停用/凍結/維護中/進行中/已結束/未開始/待審核/已通過/已拒絕）
- **驗證規則**: 預設選項，無需驗證
- **頁面**: 幾乎所有頁面（17 個頁面）

### 3. 公會 (Guild)
- **權重**: 3
- **輸入限制**: 下拉選單或文字搜尋（1-50 字元）
- **驗證規則**: 文字搜尋時無特殊限制
- **頁面**: 用戶查詢、公會列表、公會提領、公會操作記錄、公會貢獻查詢、紅包審核

### 4. 暱稱 (Nickname)
- **權重**: 4
- **輸入限制**: 1-16 字元
- **驗證規則**: `.{1,16}`
- **頁面**: 用戶查詢、交易紀錄查詢、投注紀錄查詢、會員支付查詢

### 5. 手機號碼 (Phone)
- **權重**: 5
- **輸入限制**: 09 開頭共 10 碼
- **驗證規則**: `^09\d{8}$`
- **頁面**: 用戶查詢

### 6. 註冊來源 (Registration Source)
- **權重**: 6
- **輸入限制**: 下拉選單（手機/LINE/Telegram/OpenXin）
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢

### 7. 是否禁言 (Muted)
- **權重**: 7
- **輸入限制**: 下拉選單（是/否/全部）
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢

### 8. VIP 等級 (VIP Level)
- **權重**: 8
- **輸入限制**: 下拉選單（VIP 1-6/全部）或 VIP 等級以上/VIP 等級限制
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢、活動設定、任務設定

### 9. 遊戲種類/遊戲名稱/遊戲類型 (Game)
- **權重**: 9
- **輸入限制**: 下拉選單或文字搜尋（1-50 字元）
- **驗證規則**: 文字搜尋時無特殊限制
- **頁面**: 投注紀錄查詢、遊戲管理相關頁面

### 10. 幣別 (Currency)
- **權重**: 10
- **輸入限制**: 下拉選單（金幣/星幣/全部）
- **驗證規則**: 預設選項
- **頁面**: 活動設定、任務設定、道具查詢、交易紀錄

### 11. 訂單編號 (Order ID)
- **權重**: 11
- **輸入限制**: 4-30 字元，英數字、底線、連字號
- **驗證規則**: `^[a-zA-Z0-9_-]{4,30}$`
- **頁面**: 會員支付查詢、交易紀錄查詢

### 12. 紅包訂單編號 (Red Envelope Order ID)
- **權重**: 11（與訂單編號同級）
- **輸入限制**: 4-30 字元，英數字、底線、連字號
- **驗證規則**: `^[a-zA-Z0-9_-]{4,30}$`
- **頁面**: 紅包審核、紅包紀錄

### 13. 支付類型/交易類型 (Payment/Transaction Type)
- **權重**: 12
- **輸入限制**: 下拉選單（儲值/提領/轉帳/兌換/購買/派彩/投注/返水）
- **驗證規則**: 預設選項
- **頁面**: 會員支付查詢、交易紀錄查詢

### 14. 時間範圍 (Time Range)
- **權重**: 13（最低優先，放在最後）
- **輸入限制**: 日期時間選擇器，格式 YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm
- **驗證規則**: 
  - 日期格式 `^\d{4}-\d{2}-\d{2}$`
  - 時間格式 `^\d{2}:\d{2}$`
  - 開始日期不可大於結束日期
- **頁面**: 幾乎所有有查詢功能的頁面
- **子類型**:
  - 註冊時間
  - 登入時間
  - 投注時間
  - 交易時間
  - 操作時間
  - 申請時間
  - 派彩時間
  - 開始時間/結束時間

## 完整權重排序（依優先級）

1. **帳號** (Account)
2. **狀態** (Status / 領取狀態 / 訂單狀態 / 即時開關)
3. **公會** (Guild / 角色 / 娛樂城 / 目標廳 / 供應商 / 付款方式 / 付款通道)
4. **暱稱** (Nickname / 發言人 / 禁用語 / 活動名稱 / 標題 / 操作者 / 申請人)
5. **手機號碼** (Phone)
6. **註冊來源** (Registration Source / 貢獻來源)
7. **是否禁言** (Muted)
8. **VIP 等級** (VIP Level)
9. **遊戲種類** (Game / 道具類型 / 維護類型 / 類型 / 種類 / 分類)
10. **幣別** (Currency)
11. **訂單編號** (Order ID / 紅包訂單編號 / 注單號碼 / 序號群組編號 / 清單編號)

13. **時間範圍** (Time Range / 內容 / 備註)

## 使用說明

1. **所有頁面的篩選欄位必須按照此權重順序排列**
2. **權重數字越小，優先級越高，應放在越前面**
3. **同一權重級別的欄位，按照業務邏輯排列（如帳號在暱稱前）**
4. **時間範圍永遠放在最後**
5. **所有輸入欄位必須實作對應的驗證規則**
6. **驗證錯誤訊息應顯示在欄位下方的 `.filter-error` 區域**

## 驗證實作範例

```javascript
// 帳號驗證
case 'account':
  const accounts = val.split(',').map(s => s.trim()).filter(s => s);
  for (const acc of accounts) {
    if (!/^[a-zA-Z0-9_]{2,20}$/.test(acc)) {
      errorMsg = '帳號格式：2~20字元英數底線';
      break;
    }
  }
  break;

// 暱稱驗證
case 'nickname':
  if (val.length < 1 || val.length > 16) {
    errorMsg = '暱稱長度：1~16字元';
  }
  break;

// 手機驗證
case 'phone':
  if (!/^09\d{8}$/.test(val)) {
    errorMsg = '格式：09開頭共10碼';
  }
  break;

// 訂單編號驗證
case 'order':
  if (!/^[a-zA-Z0-9_-]{4,30}$/.test(val)) {
    errorMsg = '編號格式：4~30字元英數';
  }
  break;

// 金額驗證
case 'amount':
  if (!/^\d+(\.\d{1,2})?$/.test(val)) {
    errorMsg = '請輸入有效金額';
  }
  break;

// 日期驗證
case 'date':
  if (val.includes('~')) {
    const parts = val.split('~').map(s => s.trim());
    const dateRe = /^\d{4}-\d{2}-\d{2}$/;
    if (!dateRe.test(parts[0]) || !dateRe.test(parts[1])) {
      errorMsg = '格式：YYYY-MM-DD ~ YYYY-MM-DD';
    } else if (parts[0] > parts[1]) {
      errorMsg = '開始日期不可大於結束日期';
    }
  }
  break;
```
