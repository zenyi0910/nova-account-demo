# Nova 後台篩選欄位權重與輸入限制規範

## 欄位產生規則

### UUID（主鍵 / ID）
- **格式**: UUIDv7（GUID）
- **範例**: `0193a2b4-8f4a-7c1d-9e2f-3a4b5c6d7e8f`
- **長度**: 36（含 `-`）
- **字元集**: `[0-9a-f-]`

### Account（會員帳號）
- **手動註冊**: 6–20 字元，英文字母 + 數字，不含特殊符號
- **自動產生**: 3 碼英文 + 4 碼數字（共 7 字元），範例 `AbC1234`
- **後台帳號**: 3–20 字元
- **驗證規則**: `^[a-zA-Z0-9]{3,20}$`

### Nickname（暱稱）
- **長度**: 2–14 字元
- **自動產生格式**: 前綴 + 後綴 + 4 碼數字
  - 前綴：玩家/用戶/會員/新手/高手/大師/俠客/遊俠
  - 後綴：001/888/999/520/168/666
  - 範例：`玩家8883456`

### PhoneNumber（手機號碼）
- **類型**: 台灣手機號碼
- **儲存格式**: `09XXXXXXXX`（10 字元）
- **支援輸入格式**:
  - `0912345678`
  - `886912345678`
  - `+886912345678`
- **驗證規則**: `^(\+?886|0)9\d{8}$`

### OrderNumber（訂單編號）
- **長度**: 21 字元
- **格式**: 前綴(3) + 時間戳(14, yyyyMMddHHmmss) + 隨機數(4)
- **字元**: 大寫英文 + 數字
- **類型前綴**:
  - 儲值: `DEP`（範例 `DEP202605071030001234`）
  - Topup: `TOP`（範例 `TOP202605071030005678`）
  - 簽到卡: `CIC`（範例 `CIC202605071030009012`）
  - 支付: `PAY`（範例 `PAY202605071030001111`）
- **驗證規則**: `^[A-Z]{3}\d{18}$`

### RedEnvelopeId（紅包編號）
- **長度**: 12 字元
- **格式**: 日期(yyyyMMdd) + 4 碼隨機數
- **範例**: `202605071234`
- **驗證規則**: `^\d{12}$`

### SerialNumber（序號）
- **長度**: 6 字元
- **字元**: 英數混合（至少 1 大寫 + 1 小寫 + 1 數字）
- **範例**: `aB3xY7`
- **驗證規則**: `^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)[A-Za-z0-9]{6}$`

### SerialNumberGroupId（序號群組）
- **格式**: UUIDv7
- **長度**: 36 字元
- **字元**: `[0-9a-f-]`

### Password（密碼）
- **長度**: 8–20 字元
- **組成**: 大寫字母 + 小寫字母 + 數字

### Code（驗證碼）
- **長度**: 6 位數字
- **類型**: 一次性驗證碼（OTP）

## 權重順序（由高到低）

### 1. 帳號 (Account)
- **權重**: 1（最高優先）
- **輸入限制**: 3-20 字元，英文字母+數字（手動註冊 6-20，自動產生 7，後台 3-20）
- **驗證規則**: `^[a-zA-Z0-9]{3,20}$`（單個帳號），可用半形逗號分隔多個帳號
- **頁面**: 用戶查詢、會員支付查詢、交易紀錄查詢、投注紀錄查詢、帳號設定

### 2. 狀態 (Status)
- **權重**: 2
- **輸入限制**: 下拉選單（啟用/停用/凍結/維護中/進行中/已結束/未開始/待審核/已通過/已拒絕）
- **驗證規則**: 預設選項，無需驗證
- **頁面**: 幾乎所有頁面（17 個頁面）

### 3. 公會 (Guild)
- **權重**: 3
- **輸入限制**: 下拉選單或文字搜尋（1-50 字元）
- **驗證規則**: 文字搜尋時無特殊限制
- **頁面**: 用戶查詢、公會列表、公會提領、公會操作記錄、公會貢獻查詢、紅包審核

### 4. 暱稱 (Nickname)
- **權重**: 4
- **輸入限制**: 2-14 字元
- **驗證規則**: `.{2,14}`
- **頁面**: 用戶查詢、交易紀錄查詢、投注紀錄查詢、會員支付查詢

### 5. 手機號碼 (Phone)
- **權重**: 5
- **輸入限制**: 台灣手機，支援 09/886/+886 開頭
- **驗證規則**: `^(\+?886|0)9\d{8}$`
- **頁面**: 用戶查詢

### 6. 註冊來源 (Registration Source)
- **權重**: 6
- **輸入限制**: 下拉選單（手機/LINE/Telegram/OpenXin）
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢

### 7. 是否禁言 (Muted)
- **權重**: 7
- **輸入限制**: 下拉選單（是/否/全部）
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢

### 8. VIP 等級 (VIP Level)
- **權重**: 8
- **輸入限制**: 下拉選單（VIP 1-6/全部）或 VIP 等級以上/VIP 等級限制
- **驗證規則**: 預設選項
- **頁面**: 用戶查詢、活動設定、任務設定

### 9. 遊戲種類/遊戲名稱/遊戲類型 (Game)
- **權重**: 9
- **輸入限制**: 下拉選單或文字搜尋（1-50 字元）
- **驗證規則**: 文字搜尋時無特殊限制
- **頁面**: 投注紀錄查詢、遊戲管理相關頁面

### 10. 幣別 (Currency)
- **權重**: 10
- **輸入限制**: 下拉選單（金幣/星幣/全部）
- **驗證規則**: 預設選項
- **頁面**: 活動設定、任務設定、道具查詢、交易紀錄

### 11. 訂單編號 (Order ID)
- **權重**: 11
- **輸入限制**: 21 字元固定長度，前綴(3碼大寫)+時間戳(14碼)+隨機數(4碼)
- **驗證規則**: `^[A-Z]{3}\d{18}$`
- **類型前綴**: DEP(儲值)/TOP(Topup)/CIC(簽到卡)/PAY(支付)
- **頁面**: 會員支付查詢、交易紀錄查詢

### 12. 紅包訂單編號 (Red Envelope Order ID)
- **權重**: 11（與訂單編號同級）
- **輸入限制**: 12 字元，日期(yyyyMMdd)+4碼隨機數
- **驗證規則**: `^\d{12}$`
- **頁面**: 紅包審核、紅包紀錄

### 13. 支付類型/交易類型 (Payment/Transaction Type)
- **權重**: 12
- **輸入限制**: 下拉選單（儲值/提領/轉帳/兌換/購買/派彩/投注/返水）
- **驗證規則**: 預設選項
- **頁面**: 會員支付查詢、交易紀錄查詢

### 14. 時間範圍 (Time Range)
- **權重**: 13（最低優先，放在最後）
- **輸入限制**: 日期時間選擇器，格式 YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm
- **驗證規則**: 
  - 日期格式 `^\d{4}-\d{2}-\d{2}$`
  - 時間格式 `^\d{2}:\d{2}$`
  - 開始日期不可大於結束日期
- **頁面**: 幾乎所有有查詢功能的頁面
- **子類型**:
  - 註冊時間
  - 登入時間
  - 投注時間
  - 交易時間
  - 操作時間
  - 申請時間
  - 派彩時間
  - 開始時間/結束時間

## 補充欄位（未在原列表中）

### 15. 序號 (Serial Number)

- **輸入限制**: 6 字元，英數混合（至少 1 大寫 + 1 小寫 + 1 數字）
- **驗證規則**: `^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)[A-Za-z0-9]{6}$`

1. **帳號** (Account)
2. **狀態** (Status / 領取狀態 / 訂單狀態 / 即時開關)

## 使用說明

1. **所有頁面的篩選欄位必須按照此權重順序排列**
2. **權重數字越小，優先級越高，應放在越前面**
3. **同一權重級別的欄位，按照業務邏輯排列（如帳號在暱稱前）**
4. **時間範圍永遠放在最後**
5. **所有輸入欄位必須實作對應的驗證規則**
6. **驗證錯誤訊息應顯示在欄位下方的 `.filter-error` 區域**

## 驗證實作範例

```javascript
// 帳號驗證
case 'account':
  const accounts = val.split(',').map(s => s.trim()).filter(s => s);
  for (const acc of accounts) {
    if (!/^[a-zA-Z0-9]{3,20}$/.test(acc)) {
      errorMsg = '帳號格式：3~20字元英數';
      break;
    }
  }
  break;

// 暱稱驗證
case 'nickname':
  if (val.length < 2 || val.length > 14) {
    errorMsg = '暱稱長度：2~14字元';
  }
  break;

// 手機驗證
case 'phone':
  if (!/^(\+?886|0)9\d{8}$/.test(val)) {
    errorMsg = '格式：09/886/+886開頭台灣手機';
  }
  break;

// 訂單編號驗證
case 'order':
  if (!/^[A-Z]{3}\d{18}$/.test(val)) {
    errorMsg = '編號格式：3碼大寫+18碼數字（共21字元）';
  }
  break;

// 紅包編號驗證
case 'redenvelope':
  if (!/^\d{12}$/.test(val)) {
    errorMsg = '紅包編號：12碼數字（yyyyMMdd+4碼隨機）';
  }
  break;

// 序號驗證
case 'serial':
  if (!/^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)[A-Za-z0-9]{6}$/.test(val)) {
    errorMsg = '序號：6字元英數混合（需含大小寫+數字）';
  }
  break;

// UUID 驗證
case 'uuid':
  if (!/^[0-9a-f]{8}-[0-9a-f]{4}-7[0-9a-f]{3}-[0-9a-f]{4}-[0-9a-f]{12}$/.test(val)) {
    errorMsg = 'UUID 格式不正確';
  }
  break;

// 金額驗證
case 'amount':
  if (!/^\d+(\.\d{1,2})?$/.test(val)) {
    errorMsg = '請輸入有效金額';
  }
  break;

// 日期驗證
case 'date':
  if (val.includes('~')) {
    const parts = val.split('~').map(s => s.trim());
    const dateRe = /^\d{4}-\d{2}-\d{2}$/;
    if (!dateRe.test(parts[0]) || !dateRe.test(parts[1])) {
      errorMsg = '格式：YYYY-MM-DD ~ YYYY-MM-DD';
    } else if (parts[0] > parts[1]) {
      errorMsg = '開始日期不可大於結束日期';
    }
  }
  break;
```
