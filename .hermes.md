# Nova 後台 Demo — Hermes 開發規範

## 🚨 最重要規則：必須使用共用元件

本專案有完整的共用元件系統，**所有後台頁面必須使用**，禁止重複造輪子。

## 必讀文件（每次修改頁面前先讀）
1. `components/README.md` — 共用元件完整 API 文檔
2. `js/components.js` — UI 元件工具庫原始碼
3. `components/sidebar.js` — 側邊欄元件

## 每個後台頁面必須引入
```html
<nav class="sidebar" id="sidebar"></nav>
<script src="components/sidebar.js"></script>
<!-- page content -->
<script src="js/components.js"></script>
<script src="components/date-picker.js"></script>
```

## 禁止事項
- ❌ 禁止在 `<style>` 中寫 `.sidebar`、`.toggle`、`.data-table`、`.filter-card` 等共用 class 的 CSS
- ❌ 禁止自定義按鈕 class（如 `btn-export`、`btn-batch`）
- ❌ 禁止手寫 toggle HTML，必須用 `UI.toggle()`
- ❌ 禁止手寫表格 HTML，必須用 `UI.table.create()`
- ❌ 頁面 `<style>` 只允許寫該頁面專屬樣式

## 按鈕必須用 UI.btn.*
| 場景 | 正確 |
|------|------|
| 編輯排序 | `UI.btn.sort('編輯排序', 'enterEdit()')` |
| 取消 | `UI.btn.sortCancel('取消', 'exitEdit()')` |
| 儲存排序 | `UI.btn.sortSave('儲存排序', 'saveSort()')` |
| 新增 | `UI.btn.add('新增', 'openAdd()')` |
| 搜尋 | `UI.btn.search('搜尋', 'doSearch()')` |
| 編輯圖示 | `UI.btn.icon('edit', 'edit(id)')` |
| 刪除圖示 | `UI.btn.icon('delete', 'del(id)')` |
| Toggle | `UI.toggle(status, 'toggleFn(id)')` |

## 表格用法
```javascript
UI.table.create(columns, rows, { empty: '尚無資料' });
```

## Sidebar
Sidebar 的 CSS 和 HTML 完全由 `components/sidebar.js` 自動注入，頁面只需放：
```html
<nav class="sidebar" id="sidebar"></nav>
<script src="components/sidebar.js"></script>
```
不需要、也不允許在 `<style>` 裡寫任何 sidebar 相關樣式。

## 修改頁面前的檢查清單
1. 是否引入了 `js/components.js`？
2. 按鈕是否用 `UI.btn.*`？
3. Toggle 是否用 `UI.toggle()`？
4. 表格是否用 `UI.table.create()`？
5. `<style>` 是否只有頁面專屬樣式？
